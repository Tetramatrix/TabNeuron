Inspiration

- -

Most tab managers are confined to tiny browser sidebars or dropdown menus, trapping your research in flat, ephemeral lists. We built TabNeuron because complex web research requires space and permanence. By pulling your browser tabs out onto a native desktop canvas, you gain an infinite 2D workspace to visually group information, analyze it with AI, and take true ownership of your browsing sessions.

What is TabNeuron?

- -

TabNeuron solves the problem that tabs are hidden in browser bars, AI forgets everything between sessions, and web research is manual. Map your active browsing session onto an infinite visual canvas. Organize tabs spatially instead of scrolling through flat lists. Chat with websites to get instant answers without manual reading. Automate research with AI agent pipelines — scrape, analyze, and output in sequence. Sync bidirectionally with Chrome Tab Groups so your browser and workspace stay aligned. AI memory that extracts facts, skills, and preferences from your chats, making answers more accurate over time. Run AI models locally so nothing leaves your machine, or connect to cloud AI.

Core Capabilities

What TabNeuron solves — each category expands to show how it works.

+ 📂 Organize Tabs on Canvas

Problem: "Which tab has that info?" — Scrolling through dozens of hidden tabs in browser bars, losing context.

  • Solution: AI groups tabs on a 2D canvas by topic — visually find tabs by what they're about, not where they're buried.
  • Canvas Editor: Drag and drop, rename, resize — full visual control.
  • Mind Maps: Generate visual representations of tab groups to reveal hidden relationships.

+ 🔄 Canvas ↔ Browser Sync

Problem: Workspace and browser are disconnected — changes in one don't reflect in the other.

  • Solution: Bidirectional sync with Chrome Tab Groups — canvas changes push to browser, browser changes pull to canvas.
  • Workspace → Browser: AI suggests semantic groupings, you review and click "Apply" — browser tab groups update instantly.
  • Browser → Workspace: Create groups or open tabs in browser — TabNeuron's polling API detects changes and pulls them onto the canvas.

+ 💬 Chat with Websites

Problem: Opening and reading across multiple tabs takes too long. Research means comparing, copying, synthesizing manually.

  • Solution: Select multiple tabs and ask the AI to synthesize, compare, or extract data across all of them simultaneously.
  • DOM Extraction: Page content and metadata fed directly into LLM context — no manual reading.
  • Multi-Tab Chat: "Compare specs across 4 review tabs" → result saved locally.

+ 🤖 AI Agent Pipelines

Problem: Repetitive web research is manual — open tabs, extract data, compare, format, save — every time.

  • Solution: Build no-code automated workflows where one agent's output becomes the next agent's input.
  • Scraper Agent: Reads selected tabs, uses web_extract_links to find specs.
  • Analyst Agent: Takes output, compares, formats a Markdown table.
  • Output Agent: Writes executive summary, saves as persistent text node on canvas.

+ 📸 Visual Previews & Metadata

Problem: Can't tell what a tab contains without switching to it — title alone isn't enough.

  • Solution: Automatic scaled screenshots, titles, URLs, and OG-tags extracted for every tab — instant visual context.
  • Data Ownership: Save workspace states locally to disk. Close browser, research session remains safely backed up.

+ 🧠 AI Memory That Remembers

Problem: AI forgets who you are after every session. Long chats are expensive and slow.

  • Solution: Auto-extracts facts, skills, preferences, and personas from conversations. Gets better over time.
  • Auto-Compaction: Old conversations summarized → 93% token savings, full context preserved.
  • Global Memory: Shared across all agents, import/export, fully manageable.

+ 🔌 MCP Server & Integrations

Problem: Repetitive file tasks are manual. Web research means opening multiple tabs, comparing, copying. Research takes forever.

  • MCP Manager: Visual interface — enable/disable servers, configure settings, monitor health.
  • File Operations: read, write, move, search, edit — all through conversation.
  • Web Tools: Fetch, scrape, search, extract — AI has direct web access.
  • External Servers: Connect Google Drive, GitHub, PostgreSQL, custom tools.

+ ☁️ AI Backends — Local or Cloud

Problem: Data goes to cloud providers — no privacy. Or local models are hard to configure.

  • Local: 🍋 Lemonade (NPU acceleration), Ollama, Llama.cpp, LM Studio — nothing leaves your machine.
  • Cloud: OpenAI, Mistral, Gemini (free tier), and more — when you need power.
  • Model Manager: Activate/deactivate models per task — 2-pane view (🍋 Lemonade only) for chat + embedding simultaneously.

+ 📦 Technical Details

  • Portable: Runs from USB, extract anywhere, data stays under your control.
  • Centralized Storage: All data in a single .tabneuron/ folder. Existing installs migrate automatically.
  • Browser Bridge: Lightweight extension connects via local polling API — no data leaves your machine.

🔄 Visual AI Workspace ↔ Browser Sync

- -

Your workspace and your browser stay aligned through a controlled bidirectional sync.

+ Workspace → Browser

  • Organize tabs spatially on the desktop canvas or let the AI cluster them semantically.
  • Review the layout and click "Apply".
  • Your browser's native tab groups are instantly updated to match your visual workspace.

+ Browser → Workspace

  • Create groups or open new tabs directly in your browser.
  • TabNeuron's lightweight polling API detects the changes.
  • The new tabs and groups are automatically pulled onto your desktop canvas.

Note: While our architecture is browser-agnostic, the current bridge extension only supports Chromium-based browsers (Chrome, Edge, Brave, etc.).

AI Memory & Smart Routing

- -

TabNeuron's AI remembers who you are, what you prefer, and what you've discussed — making every answer more accurate while using dramatically fewer tokens. Perfect for local and private inference.

+ Memory That Learns Over Time

Problem: AI starts from scratch every session — forgets who you are, what you do, what you prefer.

  • Active Conversation — Your current chat, always in context
  • Auto-Compaction — Long chats compressed into summaries (93% token reduction). Context stays, cost drops.
  • Observations — Facts, preferences, and skills auto-extracted from every message
  • Profiles — Observations consolidated into entity summaries (you, companies, topics) — 61% fewer tokens

+ How It Works

  • Auto-Extraction: Every message analyzed — memories saved automatically, no manual effort
  • Profile-First Retrieval: Compact profiles loaded first, detailed observations as fallback — fast and precise
  • Adaptive Document Search: Semantic search → BM25 → Keyword → Full Text — precise, fast, works offline
  • Conversation Compaction: 17,000 → 1,200 tokens while keeping full context
  • Cross-Agent Memory: Share memories between agents, import/export, global memories

+ Intent Classification & Smart Routing

Problem: AI doesn't know whether to search memory, call a tool, or query documents — wastes tokens on wrong actions.

  • Intent Classification: Detects memory search, tool calls, document queries, or hybrid intents automatically
  • Entity Extraction: Identifies companies, users, topics from natural language
  • Smart Routing: Routes to correct MCP server (file ops, web tools) — no wasted calls
  • Loop Prevention: Max 3 tool calls per request — prevents infinite loops
  • Multi-Language: English + German intent patterns supported

+ Key Benefits

  • Token Efficiency: Only relevant context loaded — profile-first retrieval, fallback only when needed. Dramatically fewer tokens, faster answers.
  • Live Session Stats: Real-time performance metrics — tokens/sec, time-to-first-token, input/output tokens — visible in every chat. (🍋 Lemonade only)
  • Global Memory: Learns from every conversation — memories shared across ALL agents, creating a unified knowledge base.
  • Memory Import/Export: Transfer memories between projects, backup and share knowledge.
  • Configurable Compaction: Control when and how conversations are compressed — perfect thresholds for your workflow.
  • NPU Acceleration: 🍋 Lemonade with FastFlowLM for native AMD Ryzen NPU acceleration. Falls back to text-based retrieval if no NPU available.
  • Multi-Language: English + German for memory routing, 25+ languages for tool intents, 15+ languages for document retrieval — all auto-detected.

MCP Server & Integrations

- -

TabNeuron solves the problem that file tasks, web research, and email management are all manual, disconnected workflows. Through MCP servers, your AI can interact with files, web content, and external services — all through conversation. Connect any third-party MCP server (Google Drive, GitHub, PostgreSQL, custom tools) for unlimited extensibility.

  1. Open MCP Manager to configure and enable servers
  2. Connect external MCP servers if needed (Google Drive, GitHub, databases, etc.)
  3. Create an agent in the workspace
  4. Right-click on the agent title and select "Chat"
  5. Interact directly with files, folders, web content, and external services

+ MCP Manager

Problem: Managing MCP servers means editing config files manually — error-prone and tedious.

Solution: Visual interface for managing MCP servers. Enable/disable servers, configure settings, and monitor connection health — all without touching config files. Connect any compatible external MCP server for unlimited extensibility.

+ File Operations

Problem: Repetitive file tasks — creating folders, moving files, reading, editing — require manual clicking and navigation.

Solution: Automate file operations through conversation:

  • "List the first 10 files in the current directory" (uses list_directory)
  • "Search all text files in the workspace" (uses search_files)
  • "Read the content of document.txt to summarize its contents" (uses read_file)
  • "Read lines 10-20 of large_log.txt to check for errors" (uses read_file_from_line)
  • "Move file report.docx to archive/report_backup.docx" (uses move_file)
  • "Get size and creation date of config.json" (uses get_file_info)
  • "Edit the third paragraph of essay.txt to improve clarity" (uses edit_file)
  • "Write the generated report to monthly_report.txt" (uses write_file)
  • "Create a new folder called 'Project_X' in the workspace" (uses create_directory)
  • "Show available storage space and supported operations" (uses get_filesystem_info)
  • "Summarize document.txt and create a markdown summary document" (uses read_file + write_file)

Document Processing & OCR

- -

Problem: Scanned PDFs and images are invisible to AI — no text content to search or chat with.

Solution: OCR extracts text from scanned documents, images, and PDFs. Once processed, documents become searchable and indexed via adaptive retrieval (semantic search → keywords → full-text fallback) so AI finds precise answers from your documents. Process documents via the context menu ('Document Overview' and 'Process Documents') — particularly valuable for PDFs and image-based documents.

+ OCR Capabilities

TabNeuron processes various document types, extracting text so AI can work with them:

  • Text PDFs: Extract text from PDF documents (supports embedded text and OCR for scanned content)
  • Plain Text Files: Process .txt files with support for:
    • UTF-8 (recommended for full Unicode support)
    • Latin-1 (ISO-8859-1) as fallback encoding
  • Code Files: OCR support for source code files including:
    • Python (.py), C++ (.cpp), JavaScript (.js), Java (.java)
    • C# (.cs), PHP (.php), Ruby (.rb), Go (.go)
    • TypeScript (.ts), Swift (.swift), Kotlin (.kt)
    • And other common programming language files in UTF-8 or Latin-1 encoding
  • PDFs with Images: Built-in method for OCR processing of PDFs containing images

+ Requirements for PDF Image OCR

To enable OCR for PDFs with images, you need to install the official Tesseract OCR engine with default settings and ensure it's available in your system PATH. Tesseract is an open-source OCR engine that provides high-quality text extraction from images.

Download Tesseract from:

Tesseract OCR GitHub

+ Encoding Support Notes

The application primarily uses UTF-8 encoding for document processing and falls back to Latin-1 (ISO-8859-1) when UTF-8 decoding fails. For optimal results, we recommend using UTF-8 encoding for your documents. This ensures the best compatibility with international characters and special symbols.

AI Backends & Models

- -

+ AI Model Performance

TabNeuron uses AI models to intelligently group tabs on the canvas and power chat, agents, and research. The built-in default model is lightweight and works offline, but may not always classify tabs perfectly, especially for less common content. For optimal performance, we strongly recommend using Llama 3.1 8b Instruct or higher models. Processing time depends entirely on the number of tabs - fewer tabs mean faster processing. Naturally, the best performance is achieved with cloud services.

+ AI Integration

TabNeuron supports connection to both on-prem and cloud-based AI services:

  • Built-in Models: Pre-configured models accessible through the model manager
  • Local and On-Prem Services: 🍋 Lemonade (with FastFlowLM NPU), Llama.cpp, Ollama, and other self-hosted LLM solutions
  • Cloud Services: OpenAI, Mistral, Google Gemini, and other cloud-based AI platforms

+ Model Manager Split View

The Model Manager features a split view when using 🍋 Lemonade as the backend. 🍋 Lemonade supports multi-model loading, allowing both a chat model and an embedding model to run simultaneously:

  • Upper Pane: Chat/Completion models (for conversation and reasoning)
  • Bottom Pane: Embedding models (for semantic search and RAG) — runs on NPU or GPU

Embedding models power the 4-tier RAG system's Tier 1 (NPU Embeddings). Without an embedding model, the system gracefully falls back: BM25 → Keyword → Full Text. Currently, only 🍋 Lemonade supports multi-model loading. Other backends (Ollama, LM Studio, etc.) do not support loading an embedding model alongside a chat model — but the 4-tier RAG system ensures full functionality even without embeddings.

+ Hardware Requirements

  • Built-in Models: Range from 1B parameter (806MB) to 20B parameter (8-16GB)
  • Recommended 8B Models (e.g., Llama 3.1 8b Instruct): Minimum 8 GB RAM or 4 GB VRAM for smooth operation
  • Hardware Requirements Increase with Model Size: Larger models with more parameters require higher specifications
  • Cloud Models: No local hardware requirements (requires internet connection)

+ Recommendation

🍋 Lemonade For optimal local and on-prem performance with both GPU and NPU acceleration, we recommend:

🍋 Lemonade ⚡ FastFlowLM

🍋 Lemonade is AMD's new high-performance AI server (C++ based) with integrated backends including Llama.cpp, Whisper, and FastFlowLM for native Ryzen NPU acceleration.

Note: Larger models can provide better tab grouping and analysis, but require more memory and processing power. Processing time is directly proportional to the number of tabs - fewer tabs result in faster processing.

  • Port: 5555
  • Sync: Polling API (default 2s interval)
